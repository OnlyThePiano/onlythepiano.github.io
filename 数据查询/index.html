数据查询

学生表：Student(Sno, Sname, Ssex, Sage, Sdept)

课程表：Course(Cno, Cname, Cpno, Ccredit)

学生选课表：SC(Sno, Cno, Grade)

Sdept：院系、Cpno：先修课、Ccredit：学分

单表查询

1.选择表中若干列

查询指定列：SELECT Sno, Sname FROM Student;

查询全部列：SELECT * FROM Student;

查询经过计算的值：SELECT Sname, 2014-Sage FROM Student;

  • select子句的<目标表达式>不仅可以是表中的属性，也可以是表达式。
  • 可以通过指定别名改变查询结果的列标题。

2.选择标准若干元组

消除重复元组：SELECT DISTINCT Sno FROM SC;

  • 如果没有指定DISTINCT关键词，默认ALL。

查询满足条件的元组：SELECT Sname FROM Student Where Sdept = ‘CS’;

  • 比较：= > < ………
  • 确定范围：between and、not between and
  • 确定集合：in 、not in
  • 字符匹配：like、not like
  • 空值：is null、is not null
  • 多重条件：and、or、not

3.ORDER BY 子句

升序：ORDER BY Grade ASC;

降序：ORDER BY Grade DESC;

4.聚集函数

COUNT(*)：统计元组个数

COUNT(<列名>)：统计一列中值得个数

SUM(<列名>)：计算列指得总和

AVG<列名>)：计算一列中的最大值

MAX(<列名>)：求一列中最大值

MIN(<列名>)：求一列中最小值

where子句不能用聚集函数作为条件表达式，聚集函数总能用GROUP BY中的HAVING子句或SELECT子句。

5.GROUP BY 子句

将查询结果按某一列或多列的值分组，值相等的一组。

各个课号相应的选课人数：SELECT Cno, COUNT(Sno) FROM SC GROUP BY Cno;

HAVING子句：查询平均分大于等于90的学生学号和平均成绩

SELECT Sno, AVG(Grade) FROM SC GROUP BY Sno Having AVG(Grade) > 90;

连接查询

1.等值与非等值连接查询

查询选修2号课程且成绩在90分以上的所有学生的学号和姓名：

SELECT Student.Sno, Sname

FROM Student, SC

WHERE Student.Sno = SC.Sno AND SC.Cno='2’ AND SC.Grade>90;

  • 过程是，先从SC中挑选出限定条件的元组，再和Student中满足连接条件的元组进行连接。

  • 一条SQL语句可以同时完成选择和连接查询。

2.自身连接

连接操作也可以和自己连接，需要为自身表取两个别名。

3.外连接

外连接：把舍弃的悬浮元组也保留到结果关系中。

左连接：保留左边关系中的悬浮元组。

  • <左表> LEFT OUTER JOIN <右表> ON <连接条件表达式> 。
  • <左表> LEFT OUTER JOIN <右表> ON USING(？) 。
  • 应该总是提供 <连接条件表达式> ，否则会得到笛卡尔积。
  • USING 去除重复值。

右连接：保留右边关系中的悬浮元组。

4.多表连接

查询每个学生的学号、姓名、选修的课程名和成绩：

SELECT Student.Sno, Sname, Cname, Grade

FROM Student, SC, Course

WHERE Student.Sno=SC.Sno AND SC.Cno=Course.Cno;

  • 可能过程，先将Student表和SC表进行连接得到每个学生的学号、姓名、成绩和选修的课程号，然后再通过课程号与Course表连接。

嵌套查询

SELECT-FROM-WHERE 语句称为查询块，将一个查询块嵌套在另一个查询块的WHERE子句或HAVING短句的条件中称为嵌套查询。

子查询的SELECT语句中不能用ORDER BY 子句。

实现一个查询请求可以有多种方法，不同方法其执行效率差别也就不一样。所以，在实际应用中，能用连接运算尽可能用连接表达。

表与表之间的联系

当表与表之间有直接联系或者间接联系时，通过这个联系可以选择连接运算或者嵌套运算。

查询与“刘晨”同一个系的学生：Student 表联系。

查询选修了1号课程的学生姓名：课程选修表（SC）和 Student 表联系。

  • SC 表：选修、1号课程
  • Student 表：学生姓名

查询选修了课程名为“信息系统”的学生姓名：SC 表、Course 表 和 Student 表联系。

  • SC 表：选修
  • Student 表：学生姓名
  • Course 表：课程名

查询选修了全部课程的学生姓名：SC 表、Course 表 和 Student 表联系。

  • 全部课程是 Course 表，因为 可能没有学生选修全部课程，这时候 SC 表只能表示部分课程。

相关子查询和不相关子查询

子查询的查询条件不依赖于父查询，不相关子查询。反之。

1.带有IN谓语的子查询

查询与“刘晨”同一个系的学生：

1)嵌套

SELECT Sno, Sname, Sdept
FROM Student
Where Sdept IN
	(SELECT Sdept
     FROM Student
     WHERE Sname = "刘晨"
    );
    
不相关子查询

2)连接

SELECT S1.Sno, S2.Sname, S1.Sdept
FROM Student S1, Student S2
WHERE S1.Sdept=S2.Sdept AND S2.Sname="刘晨";

查询选修了课程名为信息系统的学生学号和姓名：

1)嵌套

SELECT Sno， Sname
FROM Student
WHERE Sno IN
	(SELECT Sno
     FROM SC
     WHERE Cno IN
     	(SELECT Cno
         FROM Course
         WHERE Cname = "信息系统"
        )
    );
  • 涉及到的属性：学号、姓名、课程名，而学号、姓名存放在Student表，课程名存放在Course表，所以必须通过SC表建立联系。

2)连接

SELECT Student.Sno, Sname
FROM Student, SC, Course
WHERE Student.Sno = SC.Sno AND
	SC.Cno = Course.Cno AND
	Course.Cname = "信息系统";

2.带有比较运算符的子查询

又一次查找与“刘晨”同一个系的学生：可以用 = 代替 IN。

找出每个学生超过他自己选修课程平均成绩的课程号：

SELECT Sno,Cno
FROM SC x
WHERE Grade >= (
    SELECT AVG(Grade)
    FROM SC y
    WHERE y.Sno=x.Sno
);

相关子查询

相关子查询过程：

  1. 从外层查询中取出SC的一个元组x，将x的Sno值（201215121）传给内层。
  2. 内层循环得到的值，用该值代替内层查询，得到外层查询。
  3. 执行新的查询。
  4. 重复；直到外层的SC元组全部处理完毕。

4.带有EXISTS谓词的子查询

EXISTS代表存在量词。

EXISTS引出的子查询目标列表达式通常用 * ，因为其子查询只返回真值或假值。

带有 IN 谓词、比较运算符的子查询都能用带有 EXISTS 谓词的等价代替。

再一次查找与“刘晨”同一个系的学生：

IN：

SELECT Sno, Sname, Sdept
FROM Student
Where Sdept IN
	(SELECT Sdept
     FROM Student
     WHERE Sname = "刘晨"
    );
    


EXISTS：

SELECT Sno, Sname, Sdept
FROM Student S1
Where EXISTS
	(SELECT *
     FROM Student S2
     WHERE S2.Sdept=S1.Sdept AND S2.Sname = "刘晨"
    );
