目标

依赖关系和包含关系；依赖关系和关联关系时类图最常见的关系，不能区分关系就会使得一整个类图都是同一种关系，很明显这样的一个类图是，，，错的吧。

依赖关系和包含关系

  • 举例：删除操作，如果查询类改变的话，删除类会受到影响吗？查询类是独立的，但是删除类要依赖于查询类；

  • 包含关系是特殊的依赖关系；而包含关系的理解就是，将类可以独立的部分划分出去，使得两个部分成为包含关系；

依赖关系和关联关系的区分

主要是，关键是对has和use的关系的理解就可以了；

  • 关联是“HAS”关系，依赖是“USE”关系：

    ①A类依赖B类，指的是B的对象作为A类的方法参数存在，称为“use”关系。

    ②A类关联B类，指的是B类对象作为A类的属性存在，称为“has”关系。

  • 生命周期不同：

    ①如果A类依赖B类，那么只有当A类对象调用到相应方法时，B类对象才被临时创建，方法执行结束，B类对象即被回收，A类和B类之间的依赖关系是一种瞬时的关系。

    ②如果A类关联B类，那么创建A类的对象时实例化B类的对象，直到A类对象被销毁，所关联的B类对象也被销毁。即只要A类对象存在，B类对象就存在。

例子

  • 可以看的出来；购物车和订单、图书之间是“Has”的关系；因为购物车里有多个订单、多个图书；

  • 顾客和购物车之间是“Use”的关系；因为顾客是使用购物车，而非顾客里有个购物车；

总结

订单类是作为工作人员的依赖关系时，订单类里只有属性以体现只作为工作人员类方法的参数；

如下：

class 订单类{
    
    查看订单(订单类 a){
    int 订单 = a.订单号;
    int 订单日期 = a.订单日期;
    .......;
	}
}

而订单类作为购物车类关联关系时：

class 购物车类{
    
    订单类 b;
    提交订单(b.订单){
        .......;
    }
    
}

购物车类是顾客类的依赖关系时，购物车类有了方法，体现了只作为顾客类方法里获得了一个该类的实例；

如下：

class 顾客类{
    
    订购图书(){
    	new 购物车();
    	购物车.管理图书();
	}

	付款(){
    	new 购物车();
   	 	购物车.提交订单();
        
	}
	//下订单后再付款，所以是付款调用订单操作；
    //

}

当图书类是购物车类关联关系时：

class 购物车类{
    
    图书类 c;
    管理图书(c.图书名, c.图书编号){
        ...........;       
    }
    
}
