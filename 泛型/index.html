泛型

所谓泛型，就是允许在定义类、接口、方法时使用类型形参，这个类型形参将在声明变量、创建对象、调用方法时动态地指定( 即传入实际的类型参数，也可称为类型实参)。Java 5改写了集合框架中的全部接口和类，为这些接口、类增加了泛型支持，从而可以在声明集合变量、创建集合对象时传入类型实参，这就是在前面程序中看到的 List< <String> 和 ArrayList<String> 两种类型。

一、泛型类

1. 形参数量

一个形参类型：

public interface List<E>
{
    void add<E x>;
    Iterator<E> iterator();
}


//List<String> 等同于：
public interface ListString extends List
{
    //原来的E形参全部变成String类型实参
    void add(String x);
    Itreator<String> itreator;
}

两个形参类型：

public interface Map<K, V>
{
    //K,V可以作为类型使用
    V put(K key, V value)
    {
        
    }
}

通过这种方式，就解决虽然程序只定 义了一个List接口，但实际使用时可以产生无数多个List 接口，只要为E传入不同的类型实参，系统就会多出一个新的List子接口。

但是需要注意的是，泛型在物理上是不存在的………

注意：

  • 必须指出: List 绝不会被替换成 ListString， 系统没有进行源代码复制，二进制代码中没有，磁盘中没有，内存中也没有。

  • 包含泛型声明的类型可以在定义变量、创建对象时传入一个类型实参，从而可以动态地生成无数多个逻辑.上的子类，但这种子类在物理上并不存在。

**注意：**泛型通过构造器的定义方式

  • 当创建带泛型声明的自定义类，为该类定义构造器时，构造器名还是原来的类名，不要增加泛型声明。

3. 泛型的继承

  • 有传入实际类型参数时：父类的所有带有形参的地方都会被替换成实参类型；

  • 没有传入实际类型参数时：父类就默认是Object类型

4. 并不存在的泛型类

前面提到过泛型在物理上是不存在的，这里补充一下为什么。

List<String> l1 = new ArrayList<>();
List<Integer> l2 = new ArrayList<>();
//事实上，l1、l2是相等的

不管为泛型的类型形参传入哪一种类型实参， 对于Java来说，它们依然被当成同一个类处理，在内存中也只占用一块内存空间，因此在静态方法、静态初始化块或者静态变量的声明和初始化中不允许使用类型形参。

注意： 在静态方法、静态初始化块或者静态变量的声明和初始化中不允许使用类型形参

二、使用类型通配符

- 为什么要使用类型通配符 ？

当使用一个泛型类时都应该为这个泛型类传入类型参数，否则编译器会提出警告；Object 类是任何类的父类，但是我们并不能将它作为泛型的参数时却达不到可以作为任何泛型类的父类的作用，因为对于泛型而言， List<String> 对象不能被当成 List<Object> 对象使用。

所以需要 类型通配符 ？ 来匹配任何类型。

如果 Foo 是 Ba r的一个子类型，而 G 是具有泛型声明的类或接口，但是 G 并不是 G 的子类型。

public void test(List<Object> c) {}
List<String> s = new ArrayList<>();
//编译错误，List<String> 对象不能被当成 List<Object> 对象使用。 
test.(s);

改为用类型通配符就可以匹配任何类型。

public void test(List<?> c) {}

- 注意：

但是，使用类型通配符需要注意的是。对于容器，使用通配符是泛型是一个未知类型的对象，Java 不允许把对象放入一个未知类型的集合里。

List<?> list = new ArrayList<String>
//编译报错
list.add(new Object());

二、类型通配符上下限

- 为什么要设置类型通配符的上限和下限 ?

List<?> 的确可以是任何泛型 List 的父类，但是有时候只希望它代表某一类泛型 List 的父类时，就需要设置其上限和下限。

1. 类型通配符上限

? extends Number ： 只能存储Number或者是Number类型的子类数据。

2. 类型通配符下限

? super Integer : 只能存储Integer或者是Integer父类元素。

三、泛型方法

泛型方法的定义格式如下：

这个T类型形参就可以在该方法内当成普通类型使用。
static <T> void fromArrayToCollection(T[] a, Collection<T> c) {}

- 泛型方法 和 泛型类的区别：

  • 与接口、类声明中定义的类型形参不同的是，方法声明中定义的形参只能在该方法里使用，而接口、类声明中定义的类型形参则可以在整个接口、类中使用；

- 值得注意的是：

  • 泛型方法不需要显式传入实际类型参数，编译器会根据实参推断；
Collection<String> cs = new Collection<>();
//当调用方法时，编译器会推断出T时String类型
fromArrayCollection(sa, cs);
