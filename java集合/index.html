Java 集合

为了保存数量不确定的数据，以及保存具有映射关系的数据(也被称为关联数组)，Java 提供了集合类。

集合类主要负责保存、盛装其他数据，因此集合类也被称为容器类。所有的集合类都位于 java.util 包下。

集合类和数组不一样，数组元素既可以是基本类型的值，也可以是对象(实际上保存的是对象的引用变量)；而集合里只能保存对象(实际上只是保存对象的引用变量)。

集合概览

Java集合大致可分为Set、List、Queue 和Map 四种体系，其中 Set 代表无序、不可重复的集合; List 代表有序、重复的集合；而 Map 则代表具有映射关系的集合，Java5 又增加了 Queue 体系集合，代表一种队列集合实现。

- Java 集合类主要由两个接口派生而出：

  • Collection
  • Map

Collection 和 Iterator 接口

Collection 接口是 List、Set 和 Queue 接口的父接口。

Iterator 必须依赖于 Collection 对象，本身不提供盛装对象的能力。

- Collection 接口定义的方法：

  1. boolean add(Object o)：向集合里添加一个元素
  2. boolean addAll(Collection c)：向集合里添加集合c的所有元素
  3. boolean remove(Object o)：删除集合中的指定元素
  4. boolean contains(Object o)：返回集合里是否包含指定元素。
  5. boolean isEmpty()：返回集合是否为空。当集合长度为0时返回true
  6. boolean retainAll(Collection c)：从集合中删除集合c里不包含的元素，交集
  7. Iterator iterator()：返回一个Iterator 对象，用于遍历集合里的元素。
  8. void clear()：清楚集合元素、集合长度变为 0
  9. int size()：该方法返回集合里元素的个数。
  10. Object[] toArray()：该方法把集合转换成一个 数组，所有的集合元素变成对应的数组元素。

所有 Collection 实现类都重写了 toString() 方法，所以可以 println() 方法一次性输出集合中的所有元素，但是怎么依次访问集合的每个元素呢？

1.使用 Lambda 表达式

- iterator 接口里定义的方法：

  • boolean hasNext()：如果被迭代的集合元素还没有被遍历完，则返回true
  • Object next()：返回集合里的下一个元素
  • void remove()：删除集合里上一次 next() 方法返回的元素
  • void forEachRemaining(Consumer action)：这是 Java 8 为 Iterator 新增的默认方法，该方法可使用Lambda表达式来遍历集合元素

Iterator 必须依赖于 Collection 对象，本身不提供盛装对象的能力。

注意： 使用迭代器当使用 Iterator 迭代访问 Collection 集合元素时，集合里的元素不能被改变，只有通过 Iterator 的 remove() 方法删除上一次next() 方法返回的集合元素才可以。

Iterator it = books.iterator();
while (it.next())
{
    String book = (String)it.next();
    books.remove(book);				//引发异常，迭代过程修改了集合元素
}

- 使用 Lambda 表达式遍历 Iterator：

public static void main()
{
    Iterator it = books.iterator();
    it.forEachRemaining(obj -> System.out.println(obj));
}

2.使用 foreach 循环遍历集合元素

同样，当使用foreach循环迭代访问集合元素时，该集合也不能被改变。

public static void main()
{
    for (Object obj : books)
    {
        String book = (String)obj;
    }
}
