Java 输入输出

File类

File 类是 java.io 包下代表与平台无关的文件和目录，也就是说,如果希望在程序中操作文件和目录，都可以通过File类来完成。

File 不能访问文件内容本身。如果需要访问文件内容本身，则需要使用输入/输出流。

1.访问文件和目录

- 访问文件名相关的方法：

  • String getName(): 返回此File对象所表示的文件名或路径名( 如果是路径，则返回最后一级子路径名)。
  • String getPath(): 返回此File 对象所对应的路径名。

- 文件检查相关的方法：

  • boolean exists(): 判断File对象所对应的文件或目录是否存在。
  • boolean isFile0: 判断File 对象所对应的是否是文件，而不是目录。
  • boolean isDirectory(): 判断File对象所对应的是否是目录，而不是文件。

- 获取文件信息：

  • long lasModified(): 最后修改的时间。
  • long length(): 文件内容长度。

- 文件操作：

  • boolean createNewFile(): 当此File对象所对应的文件不存在时，该方法将新建一个该File对象所指定的新文件。
  • boolean delete(): 删除File对象所对应的文件或路径。

- 目录操作：

  • boolean mkdir(): 试图创建一个File 对象所对应的目录。
  • String[] list(): 列出 File 对象的所有子文件名和路径名，返回String数组。
  • File[] listFiles(): 列出 File 对象的所有子文件和路径，返回File数组。
  • static File[] listRoots(): 列出系统所有的根路径。这是一个静态方法，可以直接通过File 类来调用。

2.文件过滤器

在 File 类的 list() 方法中可以接收一个 FilenameFilter 参数,通过该参数可以只列出符合条件的文件。

Java IO 流

1.流的分类

- 输入输出流：

  • 输入流是带有数据的”水管“

  • 输出流是“空水管”

  • 输入输出流的划分是从程序运行所在的内存考虑的。

- 字节流和字符流：

  • 字节流操作的数据单元是 8 位字节。
  • 字符流操作的是 16 位字符。

- 节点流和处理流：

  • 可以从向一个特定的 IO 设备(如磁盘、网络)读/写数据的流，称为节点流，节点流也被称为低级流。
  • 处理流则用于对一个已存在的流进行连接或封装，通过封装后的流来实现数据读/写功能。处理流也被称为高级流。

Java 使用处理流来包装节点流是一种典型的装饰器设计模式，通过使用处理流来包装不同的节点流，既可以消除不同节点流的实现差异，也可以提供更方便的方法来完成输入/输出功能。因此处理流也被称为包装流。

字节流和字符流

计算机的文件常被分为文本文件和二进制文件两大类一所有 能用记事本打开并看到其中字符内容的文件称为文本文件，反之则称为二进制文件。但实质是，计算机里的所有文件都是二进制文件，文本文件只是二进制文件的一种特例，当二进制文件里的内容恰好能被正常解析成字符时，则该二进制文件就变成了文本文件。

如果进行输入/输出的内容是文本内容，则应该考虑使用字符流；如果进行输入/输出的内容是二进制内容，则应该考虑使用字节流。

1.输入流

InputStream 和 Reader 是所有输入流的抽象基类。

”输入设备是水管，要读取水管“。

- 输入流的方法（字节是byte[]，字符是char[]）：

  • int read(): 读取单个字节，返回的数据可以直接转换为 int 类型。
  • int read(byte[] b): 最多读取 b.length 个字节的数据，并存储在数组 b中，返回实际读取的字节（字符）数。
  • int read(byte[] b, int off, int len): 从 off 开始读取数据。

- 乱码：

数组长度为 1024 字节时程序只需要执行一次 read() 方法 即可读取全部内容。但如果创建较小长度的字节数组，程序运行时在输出中文注释时就可能出现乱码；这是因为本文件保存时采用的是 GBK 编码方式，在这种方式下，每个中文字符占 2 字节，如果 read() 方法读取时只读到了半个中文字符，这将导致乱码。

- 关闭输入流：

与 JDBC 编程一样的，程序里打开的文件源不属于内存里的资源，垃圾回收机制无法回收该资源，因该显式关闭文件 IO 资源。

Java 7 改写了所有的 IO 资源类，它们都实现了 AutoCloseable 接口，因此都可通过自动关闭资源的try语句来关闭这些 I0 流。下面程序使用 FileReader 来读取文件身。

try{
    FileReader fr = new FileReader("test.java");
    char[] cbuff = new char[32];
    int hasRead = 0;
    
    //多次 read 才能完全读取输入流全部数据
    //遇到中文字符可能乱码
    while ((hasRead=fr.read(cbuff)) > 0)
    {
        //输出
    }
}catch (IOexception ex)
{
    //try语句自动关闭文件输入流
    ex.printStackTrance();
}

2.输出流

OutputStream 和 Writer 是所有输入流的抽象基类。

”输出设备是水管，要写入空水管“。

- 输出流的方法：

  • void write(int c): 将指定的字节/字符输出到输出流中，其中 c 既可以代表字节，也可以代表字符。

  • void write(byte[]/char[] buf): 将字节数组/字符数组中的数据输出到指定输出流中。

  • void write(byte[]/char[] buf, int off, int len): 将字节数组/字符数组中从off位置开始，长度为len的字节/字符输出到输出流中。

  • void write(String str): 将 str 字符串里包含的字符输出到指定输出流中。

  • void write(String str, int off, int len): 将str字符串里从off位置开始，长度为 len 的字符输出到指定输出流中。

3.处理流

为了简化编程。

处理流执行效率更高。

- 处理流之 PrintStream 类

//包装输出流
PrintStream ps = new PrintStream(new FileOutoutStream("test2.java"));

//实用的print方法
ps.println();
ps.print();

- 处理流之 BufferedReader 类

//包装输入流
BufferedReader bs = new BufferedReader(new FileReader("test.java"));

//实用的readLine 方法
bs.readLine();

读取其他进程数据

- 与其子进程通信：

  • InputStream getInputStream(): 获取子进程的输入流。
  • InputStream getErrortStream(): 获取子进程的输错误流。
  • OutputStream getOutputStream(): 获取子进程的输出流

如果试图让子进程读取程序中的数据，那么应该用输入流还是输出流？不是输入流，而是输出流。

要站在 Java 程序的角度来看问题，子进程读取 Java 程序的数据，就是让 Java 程序把数据输出到子进程中(就像把数据输出到文件中一样，只是现在由子进程节点代替了文件节点，所以应该使用输出流。
