#不是注释吗？为什么要加上？

Bash

不加注释#：

空白：

不加注释#：

151.101.185.194 github.global-ssl.fastly.net
192.30.253.112 github.com


79949@Cocola MINGW64 ~/Desktop/新建文件夹 (2) (master)
$ git clone https://github.com/hlk-1135/squirrel.git
Cloning into 'squirrel'...
remote: Enumerating objects: 751, done.
Receiving objects:  30% (229/751), 116.01 KiB | 8.00 KiB/s

空白：



79949@Cocola MINGW64 ~/Desktop/新建文件夹 (2) (master)
$ git clone https://github.com/hlk-1135/squirrel.git
Cloning into 'squirrel'...
remote: Enumerating objects: 751, done.
Receiving objects:  47% (358/751), 1.90 MiB | 48.00 KiB/s

加注释：

#151.101.185.194 github.global-ssl.fastly.net
#192.30.253.112 github.com


79949@Cocola MINGW64 ~/Desktop/新建文件夹 (2) (master)
$ git clone https://github.com/hlk-1135/squirrel.git
Cloning into 'squirrel'...
remote: Enumerating objects: 751, done.
Receiving objects:  49% (370/751), 7.44 MiB | 1.13 MiB/s

总结：

无论是bash还是cmd，加了”#“后速度升了上去，也就是起作用了；不加#号时，速度会比空白的速度还要更慢；
